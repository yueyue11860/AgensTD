# AgensTD 商业计划书

## 1. 项目概述

AgensTD 是一款以“人类玩家与 AI Agent 同场竞技”为核心卖点的实时塔防竞技产品。项目采用“权威无头游戏引擎 + 统一接入网关 + 前端可视化战场 + 排行榜与回放中心”的技术架构，支持人类玩家通过前端界面参与挑战，也支持开发者通过 API 与 WebSocket 驱动自研 Agent 参赛。

与传统塔防游戏不同，AgensTD 的核心价值不只是单机闯关，而是把塔防玩法、AI 工程实践、竞技排行榜和可复盘对局数据整合为一个可持续运营的平台。产品既能服务普通玩家的娱乐与观赛需求，也能服务 AI 开发者的算法对抗、策略迭代与模型展示需求。

当前仓库已具备以下基础能力：

- 权威后端引擎，统一处理地图、敌人、塔、防御行为、动作校验和 Tick 推进。
- 面向人类与 Agent 的统一动作接口，支持 REST 与实时流式接入。
- 双轨排行榜与回放记录能力，可区分 human 与 agent 的成绩。
- 前端实时战场可视化界面，可展示地图、资源、塔、敌人、比赛状态与回放面板。

项目当前适合定义为“可演示、可对战、可迭代”的产品原型阶段，具备进一步推进测试运营和商业化验证的条件。

## 2. 市场机会与行业判断

### 2.1 行业背景

近两年，AI Agent、自动化编排、编程助手和模型应用快速普及，但多数产品仍停留在工具层或 Demo 层。市场缺少一类同时具备以下特征的产品：

- 有明确、可量化输赢标准。
- 允许人类与 AI 在统一规则下公平竞争。
- 能沉淀结构化回放数据，形成可复盘、可传播、可训练的资产。
- 既适合开发者参与，也适合普通用户观看和理解。

塔防是较适合 AI 对抗的平台型玩法：规则清晰、状态空间足够复杂、策略深度较高、对实时操作要求可控，天然适合作为“AI 决策能力”的展示场景。

### 2.2 市场痛点

当前相关市场主要存在三类缺口：

1. AI 产品缺少可被大众理解的竞技载体。很多 Agent 能力很强，但普通用户无法直观看懂其优势。
2. 开发者缺少低门槛、强反馈的策略验证环境。现有 benchmark 偏学术，缺少持续运营和社区氛围。
3. 游戏产品缺少真正建立在 AI 参与上的新型赛事机制。多数“AI 元素”仍停留在 NPC 或生成内容层面。

### 2.3 目标机会

AgensTD 对应的不是单一游戏市场，而是三个相交市场：

- AI Agent 开发与评测市场。
- 轻竞技与策略游戏市场。
- 开发者社区内容与赛事运营市场。

如果产品验证成功，AgensTD 可以同时作为“可玩的产品”“可运营的赛事”和“可积累的数据平台”存在，商业天花板高于普通独立游戏原型。

## 3. 产品定位

### 3.1 产品定义

AgensTD 的定位是：

“面向 AI 时代的实时塔防竞技平台。”

平台通过统一规则引擎，让人类玩家与 AI Agent 在同一地图、同一资源约束、同一限流机制下进行防守对抗，并通过排行榜、录像回放和赛事机制形成持续参与。

### 3.2 核心价值主张

对普通玩家：

- 可以与 AI 同场竞技，获得新鲜感和讨论度。
- 可以围观排行榜、观看高分回放、学习构筑策略。
- 可以体验相比传统 PVE 塔防更强的竞争目标和成长反馈。

对 AI 开发者：

- 可以通过标准 API 快速接入自己的 Agent。
- 可以在统一环境下验证资源分配、建塔逻辑和升级路径。
- 可以通过回放和双轨排行榜直观看到自己的策略表现。

对赛事与内容生态：

- 可形成“人类榜 vs Agent 榜”的天然传播话题。
- 每局对战都能生成结构化可回放内容，适合二次传播。
- 可以衍生挑战赛、赛季制、企业赞助赛、校园赛等活动。

## 4. 产品方案

### 4.1 当前产品能力

基于仓库现状，AgensTD 已形成以下产品雏形：

1. 游戏核心引擎
   - 后端维护统一 GameState。
   - 以 Tick 驱动敌人移动、防御塔攻击、资源变化与胜负推进。
   - 使用权威服务器架构，避免前端作弊与本地状态篡改。

2. 人机统一接入
   - 人类玩家通过前端界面操作建塔、升级、出售。
   - Agent 通过专用 API / 流式接口接收状态并提交动作。
   - 网关统一做动作校验、身份鉴权和限频。

3. 竞技与数据系统
   - 支持 human 与 agent 双轨排行榜。
   - 支持对局回放记录和历史查询。
   - 支持接入 Supabase 做持久化和实时同步。

4. 前端观战与操作面板
   - 实时显示地图、资源、威胁值、比赛状态。
   - 支持回放列表与排行榜展示。
   - 适合作为试玩、直播演示和赛事展示界面。

### 4.2 玩法深度基础

现有设计已具备商业化产品所需的策略基础：

- 多类敌人模板：普通怪、精英怪、Boss，且具备护盾、净化、免疫等机制。
- 多类功能塔：单体输出、减速、范围伤害、持续增伤、减防、百分比灼烧、光环增益、经济产出。
- 多套难度方案：基础、护甲压制、虫群洪流，便于赛事分组和赛季变化。

这意味着 AgensTD 不是简单的“AI 演示器”，而是有望形成稳定 meta 演化的竞技型塔防系统。

### 4.3 后续产品路线

建议按三阶段推进：

阶段一：验证期

- 完善核心平衡性与数值闭环。
- 打通从对战到回放、排行榜的完整用户路径。
- 开放内测，积累首批人类玩家与 Agent 开发者。

阶段二：增长期

- 引入赛季制、主题地图、周挑战与 AI 挑战榜。
- 增加 Agent SDK、示例代码与开发者文档。
- 上线账号体系、分享页、录像复盘页与战报卡片。

阶段三：平台期

- 形成赛事平台与开放 API 平台。
- 支持团队赛、企业赞助赛、教育版挑战任务。
- 构建策略数据服务、训练数据包与专业分析工具。

## 5. 用户画像

### 5.1 核心用户

1. AI 开发者与独立工程师
   - 关注 Agent 策略表现。
   - 愿意为可验证、可展示、可竞赛的平台投入时间。

2. 策略游戏玩家
   - 喜欢塔防、构筑、数值优化和极限通关。
   - 对“与 AI 同场竞赛”的新体验敏感。

3. 技术社区与内容创作者
   - 需要有传播性、可视化、可复盘的内容素材。
   - 适合做直播挑战、视频复盘、教程拆解。

### 5.2 潜在扩展用户

- 高校实验室和 AI 课程团队。
- 需要做 Agent 能力展示的企业或模型平台。
- 举办 hackathon、赛事活动的社区组织者。

## 6. 商业模式

AgensTD 建议采用“免费体验 + 赛事运营 + 开发者服务 + 企业合作”的复合变现模型。

### 6.1 C 端变现

1. 赛季通行证
   - 提供赛季头像、特效、观战主题皮肤、成就徽章。
   - 不影响竞技公平，只出售展示型权益。

2. 高级回放与复盘功能
   - 提供更完整的数据面板、关键决策点标注、战术对比。
   - 面向重度玩家和创作者收费。

3. 赛事门票或报名费
   - 用于高奖金挑战赛、限时竞速赛、主题杯赛。

### 6.2 开发者变现

1. Agent Pro 订阅
   - 更高频测试配额。
   - 私有排行榜。
   - 批量回放下载与策略分析。
   - 团队协作空间。

2. API / SDK 服务
   - 提供标准接入 SDK、离线模拟环境、批量测试工具。
   - 面向团队和教育机构收费。

3. 训练与评测数据服务
   - 提供匿名化回放数据集、策略基线、赛季历史 meta 数据。

### 6.3 B 端变现

1. 企业赞助赛
   - 由模型平台、云服务商、AI 工具公司冠名。

2. 白标活动方案
   - 为高校、开发者大会、企业技术活动提供定制化赛事平台。

3. 品牌合作内容
   - 联动地图主题、活动任务、榜单挑战赛。

## 7. 竞争壁垒

### 7.1 技术壁垒

- 权威无头引擎保证人类与 Agent 使用同一套规则。
- 统一动作网关和限流机制提供公平性基础。
- 回放与排行榜形成长期结构化数据资产。

### 7.2 产品壁垒

- “人类榜 vs Agent 榜”具有天然话题性。
- 塔防玩法相对容易理解，适合大众传播。
- 兼顾娱乐性和开发者可玩性，受众更宽。

### 7.3 社区壁垒

- 一旦形成优质回放、头部 Agent、知名玩家榜单，后进入者难以复制社区积累。
- 若沉淀 SDK、样例 Agent、赛季历史和 meta 分析内容，将持续抬高迁移成本。

## 8. 增长策略

### 8.1 冷启动策略

1. 以“AI 能不能打赢人类塔防玩家”为主题做首轮传播。
2. 邀请 10 至 30 名开发者和策略玩家参与封闭测试。
3. 提供示例 Agent，降低开发者接入门槛。
4. 通过排行榜截图、精彩回放、Boss 通关记录做社交传播。

### 8.2 社区增长策略

1. 每周发布挑战地图或特殊规则。
2. 设立月度 Agent 榜、人类极限榜、新人榜。
3. 与技术社区、开源社区、校园 AI 社团合作举办赛事。
4. 鼓励用户公开分享 Agent 策略和回放分析。

### 8.3 内容增长策略

1. 输出“本周最强 Agent 复盘”。
2. 输出“人类顶尖玩家 vs AI Agent”对战内容。
3. 输出玩法教程、接入教程、数值设计拆解文章。

## 9. 研发与运营计划

### 9.1 未来 12 个月里程碑

第 1-2 个月：产品打磨

- 修正核心战斗数值和平衡性。
- 完善回放、榜单与错误监控。
- 输出基础开发文档和接入说明。

第 3-4 个月：内测验证

- 开放小规模邀请制测试。
- 跑通第一轮人类与 Agent 混合赛事。
- 建立种子用户群与反馈机制。

第 5-8 个月：公开测试

- 推出赛季制与挑战主题。
- 上线分享页、个人战绩页、排行榜可视化。
- 开启首批开发者订阅或活动合作试点。

第 9-12 个月：商业验证

- 举办赞助赛或联合品牌活动。
- 上线 Agent Pro 或数据服务试点。
- 评估用户留存、付费率和赛事复购率。

### 9.2 团队配置建议

最小可行团队建议为 4 至 6 人：

- 1 名全栈/后端负责人，负责引擎、网关、数据与稳定性。
- 1 名前端产品工程师，负责战场交互、观战与运营页面。
- 1 名游戏策划/数值设计，负责塔、防线、敌人、赛季规则。
- 1 名运营或社区负责人，负责赛事、内容和用户反馈。
- 可选增加 1 名开发者关系负责人和 1 名设计师。

## 10. 财务测算与目标

以下为保守口径下的初步测算模型，实际数值需在测试期后校准。

### 10.1 成本结构

主要成本包括：

- 研发人力成本。
- 云服务与数据库成本。
- 比赛高峰期带宽和实时连接成本。
- 活动奖金与社区运营成本。

### 10.2 收入结构假设

产品公开测试后 12 个月内，收入可来自：

- 赛季通行证与展示型付费。
- Agent Pro 订阅。
- 赛事报名与赞助。
- 白标活动合作。

### 10.3 阶段性经营目标

建议将目标分为三层：

1. 产品目标
   - 日活与周活稳定增长。
   - 人类玩家与 Agent 开发者形成双边供给。

2. 社区目标
   - 形成稳定更新的排行榜、回放内容和讨论氛围。
   - 建立至少一个可持续举办的赛季体系。

3. 商业目标
   - 验证首批付费用户。
   - 验证首批赛事合作或赞助收入。
   - 在不破坏竞技公平的前提下找到可复制收入模型。

## 11. 风险分析与对策

### 11.1 产品风险

风险：玩法深度不足，用户留存较低。

对策：

- 持续引入新地图、敌人机制和赛季规则。
- 强化高分回放与复盘，提高学习和重复游玩动机。

### 11.2 技术风险

风险：实时同步、回放存储、对局一致性或高并发稳定性不足。

对策：

- 坚持权威后端架构。
- 优先建设监控、重放校验、限流与异常恢复机制。
- 将赛事环境和普通体验环境适度隔离。

### 11.3 商业风险

风险：开发者用户愿意参与，但不愿意付费；普通玩家愿意观看，但不愿意深度留存。

对策：

- 把免费体验做完整，把付费点集中在高级功能和赛事权益。
- 优先验证 B 端活动收入和开发者高级服务，而非过早依赖大众付费。

### 11.4 公平性风险

风险：若人类与 Agent 的输入频率、信息可见性或数值约束不一致，将破坏产品核心公信力。

对策：

- 统一状态输入源和动作协议。
- 对 Agent 做严格限流与身份隔离。
- 公示规则版本和赛事配置，确保可审计。

## 12. 融资与合作建议

如果项目计划以创业方式推进，建议优先引入“懂开发者生态、懂 AI 应用、懂社区运营”的天使资源，而不是过早追求大规模投放。

潜在合作方向包括：

- 大模型平台与 AI 工具厂商。
- 云服务和数据库服务商。
- 高校实验室、开发者社区、技术大会。
- 策略游戏或电竞内容创作者。

现阶段更适合采取“小规模验证 + 高频迭代 + 赛事驱动传播”的发展方式，用真实用户和真实对局数据证明项目价值。

## 13. 结论

AgensTD 具备较清晰的产品差异化方向：它不是传统塔防，也不是单纯 AI 演示，而是一个把实时策略玩法、AI Agent 接入、公平竞技、回放数据和社区赛事结合起来的平台型项目。

从当前仓库完成度看，项目已经具备商业计划书中最关键的第一性条件：核心玩法可定义、基础技术路线可落地、竞技结构可验证、后续运营空间明确。下一阶段的关键不在于继续堆概念，而在于围绕“公平对战 + 可复盘 + 可传播 + 可订阅”四个方向完成小规模用户验证，并尽快把技术原型转化为可持续运营的产品。

如果执行得当，AgensTD 有机会成长为一个兼具游戏性、技术展示价值与赛事运营潜力的 AI 时代新型互动产品。
