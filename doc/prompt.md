# 零域裁决 v0 提示词文档

## 1. 使用说明

这份文档用于给 v0 连续投喂提示词，目标是从 0 搭建一个 AI Agent 硬核 Roguelike 塔防竞赛平台的前端原型。

推荐投喂顺序：

1. 先发“全局基准提示词”
2. 再发“统一视觉约束提示词”
3. 再按页面分别发：
首页
Agent 控制台
Run 回放页
地狱赛季排行榜
难度解锁页
API 文档页
4. 如果想一步出站点，再用“完整站点总提示词”
5. 每次都建议附上“补充约束”

项目建议名称：
零域裁决

中文定位：
AI Agent 硬核 Roguelike 塔防竞赛平台

英文副标题建议：
Authoritative Roguelike Tournament for AI Agents

---

## 2. 全局基准提示词

请为一个面向 AI Agent 竞赛的硬核 Roguelike 塔防平台设计前端界面。项目核心不是角色扮演叙事，而是高复杂度规则、服务端权威模拟、可回放、可观战、可排名。用户既可能是人类玩家，也可能是调试 Agent 的开发者。

请使用 Next.js App Router、React、TypeScript、Tailwind CSS、shadcn/ui 生成页面。只做前端 UI 和交互原型，不实现真实后端。所有数据使用高质量 mock data。组件需要可复用，页面需要响应式，桌面优先，同时兼容移动端。

产品气质要求：
整体风格偏“竞赛控制台 + 科技感战术面板 + 高压系统界面”，不是传统幻想风游戏官网。
避免通用 SaaS 外观，不要做成普通后台管理系统。
视觉关键词：硬核、冷静、精密、带压迫感、可观测、可分析。
颜色方向：深石墨、军械灰、警示橙、冷蓝、少量酸性黄绿色作为状态强调。
背景不要纯色平铺，要有轻微网格、雷达扫描、数据纹理、热区图层、渐变辉光。
字体气质要更像实验室控制台或赛事系统，不要做成营销官网默认字体风格。
动画克制但明确，突出状态切换、卡片展开、时间轴推进、实时数据更新感。
图表和状态面板要丰富，尽量使用条形、时间轴、热力格、标签状态、战术卡片、统计摘要。

信息架构要围绕以下业务：
账号与 Agent 管理
Run 创建与状态查看
对局观战
回放查看
地狱难度赛季排行榜
难度解锁进度
事件日志、动作日志、快照浏览
Agent 调试和 observation 可视化

请先建立统一的 design system，包括：
颜色变量
字体层级
按钮、卡片、标签、状态徽章、表格、侧边栏、顶部导航、日志面板、资源条、时间轴组件、排行榜组件、事件流组件、地图占位组件
提供一套高质量 mock data，字段贴近游戏业务，例如 run_id、difficulty、seed、tick、resources、towers、enemies、events、actions、replay snapshots、season ranking 等。

请确保页面有明显的“游戏竞赛系统”识别度，而不是通用控制台。

## 3. 统一视觉约束提示词

在现有页面基础上继续优化视觉统一性，要求：

强化“AI Agent 竞赛平台”辨识度
增加战术面板感和模拟器监控感
减少普通后台管理页面气质
所有关键区域都要有层次感：顶部总览、主工作区、右侧状态栏、底部日志区

多使用：
状态灯
扫描线纹理
细网格背景
渐变边框
危险等级标签
资源条
赛季徽章
难度标识
时间轴刻度
局部模糊玻璃与硬边框混合

不要过度花哨，不要赛博朋克霓虹泛滥，要保持严肃、专业、竞赛级工具感。

## 4. 首页提示词

请生成这个项目的首页，定位不是大众营销页，而是“赛事入口 + Agent 调试平台首页”。

页面目标：
让用户一眼看懂这是一个 AI Agent 可接入的硬核 Roguelike 塔防竞赛系统

引导用户进入 Agent 控制台、Run 调试、回放、排行榜

首页结构建议：

顶部导航：
Logo
Docs
Console
Replay
Leaderboard
Sign in

首屏 Hero：
一句强有力标题，强调 AI Agent、Hardcore Roguelike、Authoritative Simulation
副标题解释这是一个适合 Agent 自动决策和长期规划的高复杂度塔防竞赛平台
两个主按钮：开始调试 Agent、查看地狱赛季榜

核心特性区：
服务端权威模拟
确定性回放
四档难度逐级解锁
地狱赛季总时间排名
Agent observation/action 协议
内容深度和长程规划

流程区：
Create Run
Fetch Observation
Submit Action
Server Resolve
Replay and Ranking

难度区：

排行榜预览区：

回放预览区：
展示最近几场精彩 run 的小卡片

底部 CTA：

查看 API 契约
浏览回放系统

设计要求：
首屏视觉必须有压迫感和系统感
避免传统插画风
多用数据化视觉元素，例如节点图、波次条、资源面板、仿真监视器
需要完整响应式布局。

## 5. Agent 控制台提示词

请生成一个 Agent 调试控制台页面，这是整个平台最核心的页面。它要像“比赛控制台 + 模拟器操作台”，而不是普通表单页。

页面用途：
创建 run
选择难度和种子
查看当前 observation
查看可执行动作
模拟提交 action
查看 run 状态推进
查看资源、塔、敌人、主堡状态
查看事件日志和动作日志

页面布局建议：
左侧为配置与操作区
中间为主战场概览区
右侧为状态摘要区
底部为日志和时间轴区

详细模块：

顶部总览栏：
当前 run 状态
difficulty
seed
tick
node
region
observation version
决策倒计时

左侧面板：
Create Run 表单
选择 agent version
选择 difficulty
选择官方 seed 或随机 seed
开始 run 按钮

Action Composer
action_type 下拉
target_cell
target_entity
item_id
option_index
observation_version
提交 action 按钮

中间主区域：
战场地图占位可视化，使用网格表现 18x18 或 24x24
塔、防御设施、敌人路径、入口、主堡要有视觉标记
上方可切换视图：Map、Towers、Enemies、Heatmap、Pathing

右侧状态栏：
资源卡片：金币、热量、法能、维修点
主堡状态
当前 Buff / Debuff
已建塔列表
敌人摘要
当前可执行动作数

底部区域：
事件日志
动作日志
关键结算日志
Tick 时间轴
快照节点

要用高质量 mock data 填满，不要空状态页面。

需要交互细节：
切换 difficulty 时整体摘要会变化
点击地图格子会高亮并同步到 Action Composer
点击日志项会跳转到对应 tick
状态标签颜色要明显
界面必须体现“用于调试 Agent 决策”的专业感。

## 6. 回放页面提示词

请生成一个高质量的 Run 回放页面，用于查看某一局对战的完整过程。这个页面要突出“关键快照 + 事件流”的设计，而不是单纯视频播放器。

页面定位：
比赛仲裁
Agent 调试
规则 bug 定位
确定性复现验证

页面结构建议：
顶部为 run 基本信息
中间左侧为地图回放区
中间右侧为状态面板
底部为事件流和时间轴

详细功能模块：

顶部信息：
run_id
agent version
difficulty
seed
final result
clear time
damage taken
total actions
replay integrity status

中间左侧：
地图回放舞台
播放、暂停、逐 tick 前进、逐快照跳转
速度控制 0.5x、1x、2x、4x
显示敌人路径、塔攻击范围、热区覆盖、主堡状态

中间右侧：
当前 tick 状态
资源变化
敌人剩余
塔列表
Buff / Debuff
Boss 阶段
最近 action

底部：
时间轴
关键快照标记
事件流列表
动作流列表
过滤器：仅看战斗、仅看商店、仅看事件、仅看 Boss、仅看错误

需要做出很强的“可审计性”和“可追踪性”
视觉上像专业分析工具，不像游戏直播页
可以加入侧拉面板查看某个 tick 的 observation JSON 预览，但以易读 UI 为主，不要整屏代码块。

页面内容：

Season 编号
激活状态
官方种子数量
参赛队伍数
平均全通率
最近更新时间

排名规则说明：
第一优先级：是否全部通关

第三优先级：总主堡受损量

主榜单表格：
Rank
Team
Agent Version
Completion
Total Time
Damage Taken
Actions
Trend
Replay

支持搜索、筛选、排序、分页或分段加载

种子完成详情：
每个队伍展开后可看到 Seed 1 到 Seed 5 的通关情况

右侧附加区域：
赛季分布统计

## 10. 完整站点总提示词

请直接生成一个完整的 AI Agent Roguelike 竞赛平台前端原型，包含以下页面并使用统一设计语言：

首页
Agent 控制台
Run 回放页
地狱赛季排行榜
难度解锁页
API 文档页

技术要求：
使用 Next.js App Router、React、TypeScript、Tailwind、shadcn/ui
使用 mock data
组件化拆分
响应式
有真实产品感
页面之间可以通过导航跳转

业务背景：
这是一个高难度塔防式 Roguelike 平台，供 AI Agent 接入自动决策。所有规则由服务端权威模拟，前端只负责展示、调试、观战、回放、排行榜。核心竞争力是确定性、可回放、公平评测、长程规划和高难度赛季天梯。

请重点把 Agent 控制台、回放页、排行榜做出专业度，不要只做营销首页。

整体视觉必须体现：
硬核
严肃
高信息密度
系统控制台感
赛事可信度
未来感但克制

避免普通 SaaS 后台和普通游戏官网模板感。

## 11. 每次追加的补充约束

请不要生成过度通用的后台模板。
请减少大面积空白和常规卡片宫格。
请提高信息密度，但保证层级清晰。
请增加战术面板、日志面板、时间轴、状态徽章、资源条、赛季标签等元素。
请用更有识别度的配色和背景纹理。
请优先保证桌面端体验，同时兼顾移动端。
请所有页面都使用真实感较强的 mock data，不要用 lorem ipsum。

---

## 12. 推荐投喂策略

如果你想让 v0 生成结果更稳定，建议按下面顺序来：

1. 先生成 design system 和全局布局骨架，不要一开始就要求全站细节
2. 再单独生成 Agent 控制台，因为这是识别度最高、最复杂的页面
3. 再生成 Run 回放页，强化“快照 + 事件流”特征
4. 再生成排行榜页，建立赛事系统气质
5. 最后补首页、难度页、API 文档页
6. 每一轮都把“统一视觉约束提示词”和“补充约束”一起附上，防止风格跑偏

---

## 13. 最小可行组合

如果你只想先做 MVP 前端原型，优先生成这三页：

1. Agent 控制台
2. Run 回放页
3. 地狱赛季排行榜

原因：
这三页最能体现项目差异化，也最贴近你的核心价值：服务端权威模拟、可回放、可观战、可排名。
